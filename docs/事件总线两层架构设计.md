# 事件总线两层架构设计

> V2 事件处理系统采用**数据源层（DataSource Layer）** 与**事件总线层（EventBus Layer）** 分离的分层架构，
> 实现外部事件接收与内部事件管道的解耦。

---

## 架构总览

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        数据源层 (DataSource Layer)                       │
│  负责从外部系统接收数据变更事件，通过 publisher.Publish(event) 写入 EventBus │
│                                                                         │
│  ┌──────────────────┐     ┌──────────────────────────────────┐          │
│  │  Webhook Handler  │     │  Kafka DataSource Consumer (可选) │          │
│  │  (HTTP 回调，始终) │     │  (cfg.Kafka.Enabled=true 时启动)  │          │
│  └────────┬─────────┘     └────────────────┬─────────────────┘          │
│           │                                │                            │
│           │  publisher.Publish(event)      │  publisher.Publish(event)  │
└───────────┼────────────────────────────────┼────────────────────────────┘
            │                                │
            ▼                                ▼
┌─────────────────────────────────────────────────────────────────────────┐
│                        事件总线层 (EventBus Layer)                       │
│  数据源和图数据库写入之间的中间管道层，MPSC 架构                          │
│  cfg.EventBus.Mode 决定实现："channel"（内存）或 "kafka"（持久化）       │
│                                                                         │
│  ┌──────────────────────────────────────────────────────────────┐       │
│  │           共享通道 (chan SyncEvent 或 Kafka Topic)             │       │
│  │                                                               │       │
│  │  Publisher 端 ──→ [缓冲通道/Topic] ──→ Consumer 端            │       │
│  └──────────────────────────────────────────────────────────────┘       │
│                                                                         │
│  ┌──────────────────────────────────────────────────────────────┐       │
│  │  StartConsumer → toServiceEvent → IncrementalSync → Neo4j    │       │
│  └──────────────────────────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────────────────────────┘
```

## 数据源层 (DataSource Layer)

### 职责

从外部系统接收数据变更事件，通过统一的 `publisher.Publish(event)` 方法将事件发送到事件总线层。

### 数据源类型

| 数据源 | 类型 | 启用条件 | 说明 |
|--------|------|----------|------|
| **Webhook** | HTTP 回调 | 始终可用 | 外部系统通过 HTTP POST 推送事件 |
| **Kafka** | 消息队列 | `cfg.Kafka.Enabled = true` | 从外部 Kafka Topic 消费事件 |

### 配置项

```yaml
# 数据源层 (DataSource Layer)
kafka:
  enabled: false          # V2 默认关闭，仅 Webhook 作为数据源
  brokers: ["localhost:9092"]
  topic: "sync-events"    # 外部事件 Topic（数据源 Topic）
  group_id: "network-twin"
```

**关键配置**:
- `cfg.Kafka.Enabled`: 控制是否启动 Kafka 数据源消费者
  - `true`: 启动 Kafka DataSource Consumer，从外部 Topic 消费事件
  - `false`（默认）: 仅 Webhook 作为唯一数据源

### 数据流

```
外部系统 ──HTTP POST──→ HandleWebhook ──→ publisher.Publish(event) ──→ EventBus
外部 Kafka Topic ──→ Kafka DataSource Consumer ──→ publisher.Publish(event) ──→ EventBus
```

### 代码位置

| 组件 | 文件 | 说明 |
|------|------|------|
| Webhook Handler | `internal/service/sync_service.go` | `HandleWebhook()` 方法 |
| Kafka DataSource Consumer | `internal/events/kafka_datasource.go`（V2-03 实现） | 消费外部 Kafka Topic |

## 事件总线层 (EventBus Layer)

### 职责

作为数据源和图数据库写入之间的中间管道层，采用 MPSC（Multi-Producer Single-Consumer）架构，
所有数据源写入同一个共享通道，由单一消费者串行处理。

### 实现模式

| 模式 | 配置值 | 共享通道 | 特点 |
|------|--------|----------|------|
| **Channel**（内存） | `cfg.EventBus.Mode = "channel"` | `chan SyncEvent` | 默认，V1 兼容，进程重启丢失事件 |
| **Kafka**（持久化） | `cfg.EventBus.Mode = "kafka"` | Kafka Topic | 持久化，进程重启不丢事件 |

### 配置项

```yaml
# 事件总线层 (EventBus Layer)
event_bus:
  mode: "channel"         # V2 默认 channel，生产环境可切换为 kafka
  kafka:
    brokers: ["localhost:9092"]
    topic: "sync-events"  # EventBus 内部 Topic
    group_id: "network-twin"
```

**关键配置**:
- `cfg.EventBus.Mode`: 决定 EventBus 实现
  - `"channel"`（默认）: 内存 Channel，适用于开发和测试
  - `"kafka"`: Kafka Topic，适用于生产环境

### 共享通道架构

Publisher 和 Consumer 共享同一个底层通道，与 V1 的 `NewChannelEventBus` 设计对称：

```go
// Channel 模式
pub, con := events.NewChannelEventBus(bufferSize)  // 共享 chan SyncEvent

// Kafka 模式
pub := events.NewKafkaPublisher(brokers, topic, cfg)   // 写入 Kafka Topic
con := events.NewKafkaConsumer(brokers, topic, groupID, cfg)  // 消费同一个 Topic
```

### Fallback 机制

**Fallback 仅作用于 EventBus 层**，不影响数据源层。

当 `cfg.EventBus.Mode = "kafka"` 且 Kafka 不可用时：
1. 系统自动从 Kafka EventBus 降级到内存 Channel EventBus
2. 数据源层不受影响（Webhook 和 Kafka DataSource 继续正常工作）
3. 事件通过 Channel EventBus 继续流转
4. 后台定期尝试恢复 Kafka 连接

```
EventBus Kafka 不可用:
  DataSource Layer ──→ [Kafka EventBus 降级] ──→ Channel EventBus ──→ IncrementalSync
                           ↑ Fallback 仅在此层
```

**触发条件**:
- EventBus Kafka Publisher 发送失败时触发降级
- 降级后 slog.Warn 输出日志
- 后台 tryRecover 定期探测 Kafka 连通性

### 代码位置

| 组件 | 文件 | 说明 |
|------|------|------|
| 接口定义 | `internal/events/interface.go` | `EventPublisher` / `EventConsumer` |
| Channel 实现 | `internal/events/channel.go` | `NewChannelEventBus()` |
| Kafka Publisher | `internal/events/kafka_producer.go` | `NewKafkaPublisher()` |
| Kafka Consumer | `internal/events/kafka_consumer.go`（V2-03） | `NewKafkaConsumer()` |
| Fallback | `internal/events/fallback.go`（V2-04） | `NewFallbackPublisher()` |

## 四层组合模式

数据源层和事件总线层独立配置，产生四种组合：

| # | 数据源 | EventBus | 场景 |
|---|--------|----------|------|
| 1 | Webhook only | Channel | **V2 默认**，开发/测试环境 |
| 2 | Webhook only | Kafka | 生产环境，不需要外部 Kafka 数据源 |
| 3 | Webhook + Kafka DataSource | Channel | 需要接收外部 Kafka 事件，但内部用内存 |
| 4 | Webhook + Kafka DataSource | Kafka | **全功能生产环境** |

### 配置示例

**模式 1: Webhook + Channel（默认）**

```yaml
kafka:
  enabled: false
event_bus:
  mode: "channel"
```

**模式 4: 全功能生产环境**

```yaml
kafka:
  enabled: true           # 启用 Kafka 数据源
  brokers: ["external-kafka:9092"]
  topic: "external-events"
event_bus:
  mode: "kafka"           # 使用 Kafka 事件总线
  kafka:
    brokers: ["internal-kafka:9092"]
    topic: "sync-events"
```

## 启动流程 (main.go)

```go
// 1. 初始化事件总线层（EventBus Layer）
switch cfg.EventBus.Mode {
case "kafka":
    publisher = NewKafkaPublisher(cfg.EventBus.Kafka.Brokers, ...)
    consumer  = NewKafkaConsumer(cfg.EventBus.Kafka.Brokers, ...)  // V2-03
    // Fallback: 创建 FallbackPublisher 包装（V2-04）
default: // "channel"
    publisher, consumer = NewChannelEventBus(cfg.Channel.BufferSize)
}

// 2. 初始化数据源层（DataSource Layer）
// Webhook 始终可用（通过 SyncService.HandleWebhook）
if cfg.Kafka.Enabled {
    // 启动 Kafka DataSource Consumer → publisher.Publish(event)
}
```

## 配置项作用域

| 配置项 | 作用域 | 控制对象 |
|--------|--------|----------|
| `cfg.Kafka.Enabled` | 数据源层 | 是否启动 Kafka DataSource Consumer |
| `cfg.Kafka.Brokers` | 数据源层 | 外部 Kafka 集群地址 |
| `cfg.Kafka.Topic` | 数据源层 | 外部事件 Topic |
| `cfg.EventBus.Mode` | 事件总线层 | EventBus 实现选择（channel/kafka） |
| `cfg.EventBus.Kafka.Brokers` | 事件总线层 | EventBus Kafka 集群地址 |
| `cfg.EventBus.Kafka.Topic` | 事件总线层 | EventBus 内部 Topic |
| `cfg.Channel.BufferSize` | 事件总线层 | Channel 模式缓冲区大小 |

**注意**: 数据源层和事件总线层可以使用不同的 Kafka 集群（不同 Brokers、不同 Topic）。

---

## 禁止事项

1. **禁止数据源层直接操作 EventBus 内部实现**: 数据源只能通过 `publisher.Publish(event)` 写入
2. **禁止 HandleWebhook 绕过 publisher**: 必须通过统一的 publisher 接口
3. **禁止 Fallback 影响数据源层**: Fallback 仅切换 EventBus 实现
4. **禁止 SyncService 持有独立 eventChan**: 已改造为 publisher/consumer 接口
