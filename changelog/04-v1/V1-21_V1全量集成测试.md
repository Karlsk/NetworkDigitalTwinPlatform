## 实现内容

### 1. 各模块单元测试更新和新增

#### schema/registry_test.go — 继承合并测试

| 测试 | 场景 |
|------|------|
| TestInheritance_BasicMerge | Device extends Resource，Properties 正确合并 |
| TestInheritance_Override | 子类型同名属性覆盖父类型 |
| TestInheritance_MultiLevel | A extends B extends C，拓扑序合并 |
| TestInheritance_CycleDetection | A extends B extends A → error |
| TestInheritance_NoExtends | 无 extends 的 EntityType 正常工作 |
| TestInheritance_UnknownParent | extends 不存在的父类型 → error |

#### snapshot/manager_test.go — 属性级 Diff + MetaCache + AuditLog

| 测试 | 场景 |
|------|------|
| TestLocalDiff_PropertyAdded | 属性新增字段检测 |
| TestLocalDiff_PropertyRemoved | 属性删除字段检测 |
| TestLocalDiff_PropertyModified | 属性修改检测（含 int/float64 归一化） |
| TestMetaCache_ListPerformance | List 第二次调用 < 1ms |
| TestMetaCache_CreateInvalidates | Create 后 List 可见 |
| TestMetaCache_DeleteInvalidates | Delete 后 List 不可见 |
| TestAuditLog_Record | Create/Restore/Delete 均记录审计 |
| TestAuditLog_Query | 按 Action/Snapshot 过滤 |
| TestAuditLog_FIFO | 超出 maxEntries 淘汰旧条目 |

#### assembler/assembler_test.go — 多标签节点转换

| 测试 | 场景 |
|------|------|
| TestAssemble_SingleLabel | 无继承 → `Labels: ["Device"]` |
| TestAssemble_MultiLabel | extends Resource → `Labels: ["Resource", "Device"]` |

#### connector/factory_test.go — 工厂模式

| 测试 | 场景 |
|------|------|
| TestFactory_CreateMock | 从配置创建 Mock Connector |
| TestFactory_CreateNetbox | 从配置创建 NetboxConnector |
| TestFactory_UnknownType | 未知 type 返回错误 |

### 2. E2E 测试更新 (test/e2e/e2e_test.go)

| 测试 | 场景 |
|------|------|
| TestE2E_MultiLabelNodes | Device 节点有 `:Resource:Device` 两个 Label |
| TestE2E_InheritedProperties | Device 自动拥有 Resource 基类的 status/vendor 属性 |
| TestE2E_PropertyDiff | 创建快照→修改属性→创建快照→Diff 检测属性变更 |
