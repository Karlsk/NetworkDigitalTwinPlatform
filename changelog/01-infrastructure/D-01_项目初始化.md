# D-01: 项目初始化 + 目录骨架

## 1. 任务概述

初始化 Go 项目，创建完整的目录结构和骨架文件，确保项目可以编译通过并与架构设计完全一致。这是所有后续开发工作的基础。

| 属性 | 值 |
|------|-----|
| 所属阶段 | Phase 1: 设计阶段 |
| 预估工时 | 0.5 天 |
| 前置任务 | 无 |
| 交付物 | 完整目录结构 + go.mod + Makefile + main.go 骨架 |

## 2. 详细实现步骤

### Step 1: 初始化 Go Module

```bash
mkdir network-digital-twin && cd network-digital-twin
go mod init gitlab.com/pml/network-digital-twin
```

### Step 3: 创建 `cmd/server/main.go` 骨架

```go
package main

import (
    "context"
    "log/slog"
    "os"
    "os/signal"
    "syscall"

    "gitlab.com/pml/network-digital-twin/internal/config"
)

func main() {
    // 1. 加载配置
    cfg, err := config.Load("configs/config.yaml")
    if err != nil {
        slog.Error("failed to load config", "error", err)
        os.Exit(1)
    }

    // 2. 初始化各组件 (后续任务实现)
    _ = cfg

    // 3. 启动 MCP Server (stdio)
    // TODO: V-01 实现

    // 4. Graceful shutdown
    ctx, stop := signal.NotifyContext(context.Background(), syscall.SIGINT, syscall.SIGTERM)
    defer stop()

    <-ctx.Done()
    slog.Info("shutting down...")
}
```

### Step 4: 创建 `Makefile`

```makefile
.PHONY: build test lint docker-up docker-down

build:
	go build ./...

test:
	go test ./...

test-integration:
	go test -tags=integration ./...

test-race:
	go test -race ./...

lint:
	golangci-lint run

docker-up:
	docker-compose -f deploy/docker-compose.yml up -d

docker-down:
	docker-compose -f deploy/docker-compose.yml down
```

### Step 5: 创建 `.golangci.yml`

```yaml

  enable:

    - govet

    - gosimple

  timeout: 5m
```

### Step 6: 为每个 `.go` 文件创建 `package` 声明占位

每个 `.go` 文件只需包含 `package xxx` 声明，确保 `go build ./...` 通过。

## 3. 设计原理

- **目录结构与架构设计一一对应**：`internal/` 下每个子目录对应架构中的一个核心模块，职责边界清晰
- **engine/ 目录预留**：MVP 不实现分析引擎，但提前创建目录保持结构完整，V1 阶段实现
- **testdata/golden/ 目录**：用于 Golden File 测试，存放期望输出进行回归验证
- **pkg/utils/ 与 internal/ 分离**：URI 工具函数放在 `pkg/` 因为它是通用的，可被外部引用

## 4. 验收标准

- [ ] `go build ./...` 编译通过，无错误
- [ ] `golangci-lint run` 无 Error
- [ ] 目录结构与架构设计.md "项目结构" 章节完全一致
- [ ] `go.mod` 中 module 名为 `gitlab.com/pml/network-digital-twin`
- [ ] Makefile 的 build/test/lint/docker 命令可正常执行

## 5. 注意事项

- Go 版本建议 ≥ 1.22，使用 `log/slog` 标准库做结构化日志
- `internal/engine/` 目录下的文件 MVP 阶段只放空的 package 声明
- `configs/` 目录下的 YAML 文件先创建空模板，后续任务填充
- `.gitignore` 应包含：`*.exe`, `*.yaml.snap`, `.env`, `vendor/`
- 不要在 `go.mod` 中添加尚未使用的依赖，保持最小依赖
