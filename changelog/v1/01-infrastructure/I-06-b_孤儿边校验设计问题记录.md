## 2. 问题列表

### 问题 1: Assembler 不校验 target 类型

**位置**: `internal/assembler/assembler.go` 阶段 2（L87-101）

**现状**:
阶段 2 关系推导时，只校验 `rel.From` 的 source 类型是否符合 `RelationType.Spec.Source`，不校验 `rel.To` 对应的节点类型是否符合 `RelationType.Spec.Target`。

```go
// 当前代码只检查 source
if !containsStr(rt.Spec.Source, res.Kind) {
    // ...warn and skip
}
// 没有对 target 做校验
```

**风险**:
- `testdata/mock_netbox/devices.json` 原数据中 `upstream_links` 指向 Interface URI 而非 Device URI
- CONNECTS_TO 的 target 定义为 `[Device]`，但指向 Interface 时当前代码不会报错
- 如果目标 Interface 存在于批次中，这条违规的 CONNECTS_TO 关系会被静默创建

**建议修复方案**:
1. **方案 A**: 在阶段 2 增加 target 类型校验 — 需要 uriIndex 存储 URI→Label 映射
2. **方案 B**: 在阶段 3 孤儿边校验时通过 uriIndex 反查 Label 做类型匹配

**修复影响**:
- 需要修改 `uriIndex` 数据结构: `map[string]bool` → `map[string]string`（URI → Label）
- 增加 `ValidationWarning{Type: "target_type_mismatch"}` 警告类型

### 问题 2: 无关系去重机制

**位置**: `internal/assembler/assembler.go` 阶段 2（L104-110）

**现状**:
如果同一个关系字段的值列表中有重复 URI，会生成多条完全相同的 Relation。

```go
// 当前代码: 遍历 targetURIs 直接 append，无去重
for _, targetURI := range targetURIs {
    relations = append(relations, Relation{
        Type: relSpec.RelationType,
        From: res.URI,
        To:   targetURI,
    })
}
```

**示例**:
```
输入: interfaces: ["iface:X", "iface:X"]
输出: 2 条完全相同的 HAS_INTERFACE 关系
```

**风险**:
- Neo4j 驱动实现后，重复 MERGE 可能产生重复边
- `SyncResult.RelationsCreated` 统计不准确
- 可能影响下游分析引擎的正确性

**建议修复方案**:
1. **方案 A**: 阶段 2 推导时用 `map[string]Relation` 去重，key 为 `Type+From+To`
2. **方案 B**: 阶段 3 输出前过滤重复 Relation
3. **方案 C**: 在 GraphDB 驱动层 MERGE 时天然去重（推荐作为兜底）

**修复影响**:
- 可能改变 `SyncResult.RelationsCreated` 的值
- 需要更新相关测试的预期值

### 问题 3: 不校验 `rel.From` 存在性

**位置**: `internal/assembler/assembler.go` 阶段 3（L114-127）

**现状**:
阶段 3 孤儿边校验只检查 `rel.To` 是否在 uriIndex 中，不检查 `rel.From`。

```go
// 当前代码只检查 rel.To
if !uriIndex[rel.To] {
    // orphan
}
// 没有检查 rel.From
```

**风险评估**:
- **当前安全**: `rel.From` 总是来自 `res.URI`，而 `res.URI` 在阶段 1 已经加入 uriIndex
- **未来风险**: 如果引入外部 Relation 输入（如 Webhook 增量同步中的 `delete_relation` 事件），可能产生孤儿源

**建议修复方案**:
阶段 3 增加 `rel.From` 存在性检查，与 `rel.To` 对称:

```go
if !uriIndex[rel.From] {
    warnings = append(warnings, ValidationWarning{
        Type:   "orphan_source",
        Detail: fmt.Sprintf("%s: %s → %s (source not found)", rel.Type, rel.From, rel.To),
    })
    continue
}
if !uriIndex[rel.To] {
    // existing orphan edge logic
}
```

**修复影响**:
- 增加 `ValidationWarning{Type: "orphan_source"}` 警告类型
- `SyncResult` 可能需要增加 `OrphanSourcesSkipped` 字段
