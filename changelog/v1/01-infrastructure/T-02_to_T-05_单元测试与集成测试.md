# T-05: GraphDB Neo4j 集成测试

## 1. 任务概述

使用 testcontainers-go 启动 Neo4j CE，测试 GraphDB 的全部方法（BulkCreate/Upsert/Delete/CloneDB/BuildCypher）。

| 属性 | 值 |
|------|-----|
| 所属阶段 | Phase 4: 测试阶段 |
| 预估工时 | 2 天 |
| 前置任务 | I-09, I-10, I-11, I-12 |
| 交付物 | `internal/graph/neo4j_test.go` |

## 2. 详细测试用例

### Day 1

| 用例ID | 测试内容 | 期望结果 |
|--------|---------|---------|
| TC-G01 | Ping 连接 | 无错误 |
| TC-G02 | ClearDB | 清空指定 _db 的数据 |
| TC-G03 | BulkCreate | 批量创建节点 + 关系 |
| TC-G04 | _db 隔离 | db1 和 db2 数据互不干扰 |

### Day 2

| 用例ID | 测试内容 | 期望结果 |
|--------|---------|---------|
| TC-G05 | Upsert 节点增量合并 | SET += 属性正确 |
| TC-G06 | Upsert 关系 MERGE | 幂等不重复 |
| TC-G07 | DeleteByURIs | 节点 + DETACH DELETE |
| TC-G08 | DeleteRelations | 仅删关系 |
| TC-G09 | BuildCypher 预览 | 返回合法 Cypher |
| TC-G10 | CloneDB | 完整复制 |

## 3. 设计原理

- 使用 `testcontainers-go` 启动 Neo4j CE Docker 容器
- 每个测试函数使用独立的逻辑 DB 名（避免干扰）
- 测试完成后清理容器

```go
func setupNeo4j(t *testing.T) (graph.GraphDB, func()) {
    ctx := context.Background()
    container, err := testcontainers.GenericContainer(ctx, testcontainers.GenericContainerRequest{
        ContainerRequest: testcontainers.ContainerRequest{
            Image:        "neo4j:2025.03-community",
            ExposedPorts: []string{"7687/tcp"},
            Env:          map[string]string{"NEO4J_AUTH": "neo4j/password"},
            WaitingFor:   wait.ForLog("Started."),
        },
        Started: true,
    })
    // ...
}
```

## 4. 验收标准

- [ ] 所有 10 个测试用例通过
- [ ] `go test -tags=integration -v ./internal/graph/...` 无失败

## 5. 注意事项

- testcontainers-go 需要 Docker 环境
- 测试运行较慢（启动 Neo4j 容器 ~10s），使用 `go test -timeout 5m`
- 每个测试用独立的逻辑 DB 名（如 `test_g01`, `test_g02`）
- 清理时 `ClearDB` + `docker container stop`
