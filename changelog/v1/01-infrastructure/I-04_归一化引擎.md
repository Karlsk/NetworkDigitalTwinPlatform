# I-04: 归一化引擎 (Normalizer)

## 1. 任务概述

实现 Normalizer，读取 Schema EntityType 中的 `uriTemplate`、`fieldMapping`、`normalize` 规则，将 Connector 输出的原始 Resource 进行字段标准化，产出 `NormalizedResource`（只包含节点信息，不含关系）。

## 2. 详细实现步骤

### Day 1: 核心逻辑

**文件**: `internal/normalizer/normalizer.go`

```go
package normalizer

import (
    "fmt"
    "strings"

    "gitlab.com/pml/network-digital-twin/internal/connector"
    "gitlab.com/pml/network-digital-twin/internal/schema"
    "gitlab.com/pml/network-digital-twin/pkg/utils"
)

type Normalizer struct {
    registry schema.SchemaRegistry
}

func NewNormalizer(registry schema.SchemaRegistry) *Normalizer {
    return &Normalizer{registry: registry}
}

// Normalize Resource → NormalizedResource (只包含节点信息)
func (n *Normalizer) Normalize(resource connector.Resource) (*NormalizedResource, error) {
    // 1. 获取 EntityType
    et, err := n.registry.GetEntityType(resource.Kind)
    if err != nil {
        return nil, fmt.Errorf("get entity type %s: %w", resource.Kind, err)
    }

    // 2. 复制 Properties（避免修改原始数据）
    props := make(map[string]any, len(resource.Properties))
    for k, v := range resource.Properties {
        props[k] = v
    }

    // 3. fieldMapping: 映射字段名 (源字段 → 标准字段)
    for srcField, dstField := range et.Spec.FieldMapping {
        if val, ok := props[srcField]; ok {
            props[dstField] = val
            delete(props, srcField)
        }
    }

    // 4. normalize: 标准化字段值 (pattern → replace)
    for _, rule := range et.Spec.Normalize {
        if val, ok := props[rule.Field]; ok {
            if strVal, ok := val.(string); ok {
                props[rule.Field] = strings.ReplaceAll(strVal, rule.Pattern, rule.Replace)
            }
        }
    }

    // 5. 校验 + 默认值填充
    if err := n.registry.Validate(resource.Kind, props); err != nil {
        return nil, fmt.Errorf("validate %s: %w", resource.Kind, err)
    }

    // 6. uriTemplate: 生成 URI
    uri, err := utils.GenerateURI(et.Spec.URITemplate, et.Spec.Identity.StableKeys, props)
    if err != nil {
        return nil, fmt.Errorf("generate URI for %s: %w", resource.Kind, err)
    }

    return &NormalizedResource{
        Kind:       resource.Kind,
        URI:        uri,
        Properties: props,
    }, nil
}
```

### Day 2 (半天): URI 工具函数

**文件**: `pkg/utils/uri.go`

```go
package utils

import (
    "fmt"
    "strings"
)

// GenerateURI 根据 uriTemplate 和 stableKeys 生成 URI
// template: "device:{serial_number}"
// stableKeys: ["serial_number"]
// props: {"serial_number": "SN12345", ...}
// result: "device:SN12345"
func GenerateURI(template string, stableKeys []string, props map[string]any) (string, error) {
    result := template
    for _, key := range stableKeys {
        val, ok := props[key]
        if !ok || val == nil {
            return "", fmt.Errorf("stableKey %q not found in properties", key)
        }
        strVal, ok := val.(string)
        if !ok {
            strVal = fmt.Sprintf("%v", val)
        }
        if strVal == "" {
            return "", fmt.Errorf("stableKey %q is empty", key)
        }
        result = strings.ReplaceAll(result, "{"+key+"}", strVal)
    }
    return result, nil
}
```

## 3. 设计原理

### Normalizer 只处理节点，不处理关系

- Normalizer 读取 Schema 的 `uriTemplate`/`fieldMapping`/`normalize`/`properties`
- **不读取** `relationFields`，关系字段保留在 Properties 中传递给 GraphAssembler
- 职责分离：Normalizer 负责"标准化"，GraphAssembler 负责"关系推导"

### fieldMapping 的执行顺序

1. **fieldMapping**（字段名映射）→ 2. **normalize**（字段值标准化）→ 3. **Validate**（校验+默认值）→ 4. **uriTemplate**（URI 生成）

这个顺序确保：
- 映射后的标准字段名可以被 normalize 规则处理
- 校验在映射和标准化之后，校验的是最终值
- URI 生成使用 stableKeys 对应的最终值

### URI 工具函数放在 `pkg/utils/`

- `GenerateURI` 是通用工具，不依赖任何 internal 模块
- 放在 `pkg/` 下，未来可被外部工具引用

## 4. 验收标准

- [ ] fieldMapping 正确映射（`mgmt_ip` → `management_ip`，`hw_model` → `model`）
- [ ] normalize 正确替换（hostname 中空格替换为下划线）
- [ ] URI 基于 stableKeys 正确生成（`device:SN12345`）
- [ ] required 字段缺失时返回 error
- [ ] enum 非法值时返回 error
- [ ] 默认值正确填充（status 缺失 → 填充 "Up"）
- [ ] 关系字段保留在 Properties 中（`interfaces: [...]` 不被过滤）

## 5. 注意事项

- Properties 复制后再修改，避免修改原始 Resource 的 map
- `fieldMapping` 映射后需要 `delete(props, srcField)`，防止源字段和标准字段同时存在
- `normalize` 规则的 `pattern`/`replace` 是字符串替换（`strings.ReplaceAll`），不是正则
- URI 生成时 stableKeys 的字段必须存在且非空，否则返回 error
- `Validate` 会修改传入的 map（填充默认值），这里传入的是复制后的 map，安全
